# Architecture Decision Records (ADR)

В этом разделе собраны документы, описывающие ключевые архитектурные решения проекта.

ADR помогают:
- Зафиксировать контекст и причины принятых решений
- Сохранить знания для будущих разработчиков
- Обосновать выбор архитектурных подходов

## Формат ADR

Каждый ADR содержит:

- **Status** — текущий статус (Proposed, Accepted, Deprecated, Superseded)
- **Date** — дата создания
- **Context** — контекст и проблема
- **Decision** — принятое решение
- **Rationale** — обоснование
- **Alternatives Considered** — рассмотренные альтернативы
- **Consequences** — последствия решения

## Добавление нового ADR

1. Создайте файл `docs/adr/XXX-short-name.md` (где XXX — порядковый номер)
2. Используйте шаблон из ADR-001
3. Добавьте ссылку в раздел "Список ADR" выше
4. Обновите статус других ADR, если новый ADR заменяет старый
5. Обновите `mkdocs.yml`
